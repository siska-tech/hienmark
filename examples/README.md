# Examples - テスト用タスクファイル

このフォルダには、分析モードやタグ管理機能のテスト用に生成された100個のMarkdownファイルが含まれています。

## ファイル構成

- **ファイル数**: 100個
- **命名規則**: `task-001.md` ～ `task-100.md`
- **形式**: YAML Front Matter + Markdown本文

## FrontMatterの内容

各ファイルには以下のようなタグが含まれています：

### 必須タグ
- `status`: タスクのステータス（open, inprogress, done, completed, pending, blocked, cancelled）
- `priority`: 優先度（high, medium, low）
- `assignee`: 担当者（10人の名前からランダム）
- `start_date`: 開始日（YYYY-MM-DD形式）
- `end_date`: 終了日（YYYY-MM-DD形式）
- `tags`: タグ配列（15種類のタグセットからランダム）

### オプショナルタグ（ランダムに追加）
- `due_date`: 期限日（50%の確率）
- `depends_on`: 依存タスクIDの配列（30%の確率）
- `estimate_hours`: 見積もり時間（20%の確率）
- `sprint`: スプリント名（10%の確率）
- `category`: カテゴリ（15%の確率）

## 使用目的

1. **分析モードのテスト**
   - ガントチャートの生成テスト
   - 各種グラフ（円グラフ、棒グラフ、折線グラフ）のテスト
   - ECharts統合のテスト

2. **タグ管理機能のテスト**
   - タグインデックスの生成テスト
   - タグフィルタリングのテスト
   - タグ集計のテスト

3. **パフォーマンステスト**
   - 大量データでの動作確認
   - レンダリング速度の測定

4. **データ可視化テスト**
   - 様々なステータス・優先度の組み合わせ
   - 依存関係の可視化
   - 時系列データの分析

## ファイル例

```markdown
---

priority: medium
assignee: 山田次郎

end_date: 2025-07-20
tags: ["バグ修正", "修正", "緊急"]
due_date: 2025-05-10
---

# 新機能の追加
...
```

## 生成日

2025年10月30日

## 注意事項

- これらのファイルはテスト用に生成されたダミーデータです
- 内容はランダムに生成されているため、実際のプロジェクトとは関係ありません
- 必要に応じて手動で編集・削除してください

## 使用方法

1. ワークスペースを開く
2. この `examples` フォルダをワークスペースとして選択
3. 分析モードで各種チャートを生成
4. タグ管理機能で集計やフィルタリングをテスト
