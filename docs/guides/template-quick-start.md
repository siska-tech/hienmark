# テンプレート機能クイックスタートガイド

## はじめに

このガイドでは、タグテンプレート機能の基本的な使い方を説明します。

## 1. テンプレート管理画面を開く

### 手順

1. HienMarkを起動し、ワークスペースを開きます
2. 画面上部のヘッダーにある「**テンプレート**」ボタンをクリック
3. テンプレート管理画面が表示されます

![テンプレートボタンの位置](ヘッダーの右側、「タグ管理」「タグ設定」の隣)

## 2. 最初のテンプレートを作成する

### シナリオ: バグ報告用テンプレートを作成

1. **「新規作成」ボタンをクリック**

2. **テンプレート情報を入力:**
   - テンプレート名: `bug-report`
   - 説明: `バグ報告用のテンプレート`

3. **タグを追加:**

   **タグ1: ステータス**
   - キー: `status`
   - 値: `pending`
   - 型: `文字列`
   - 「追加」ボタンをクリック

   **タグ2: 優先度**
   - キー: `priority`
   - 値: `high`
   - 型: `文字列`
   - 「追加」ボタンをクリック

   **タグ3: タイプ**
   - キー: `type`
   - 値: `bug`
   - 型: `文字列`
   - 「追加」ボタンをクリック

   **タグ4: ラベル**
   - キー: `labels`
   - 値: `bug, needs-investigation`
   - 型: `配列（カンマ区切り）`
   - 「追加」ボタンをクリック

4. **「作成」ボタンをクリック**

✅ これで最初のテンプレートが作成されました!

## 3. テンプレートを確認する

1. テンプレート一覧に戻ります
2. 作成した `bug-report` テンプレートが表示されているはずです
3. テンプレート名をクリックして展開します
4. タグの内容を確認できます

## 4. デフォルトテンプレートを設定する

新規タスク作成時に自動的に適用されるテンプレートを設定できます（将来実装予定）。

1. テンプレートを展開した状態で「**デフォルトに設定**」ボタンをクリック
2. テンプレート名の横に緑色の「デフォルト」バッジが表示されます
3. デフォルトを解除する場合は「**デフォルト解除**」ボタンをクリック

## 5. その他のテンプレート例

### 機能開発用テンプレート

```yaml
name: feature
description: 新機能開発用テンプレート
tags:
  status: planning
  priority: medium
  type: feature
  labels: [enhancement, needs-review]
```

### 作成手順:
- キー: `status`, 値: `planning`, 型: 文字列
- キー: `priority`, 値: `medium`, 型: 文字列
- キー: `type`, 値: `feature`, 型: 文字列
- キー: `labels`, 値: `enhancement, needs-review`, 型: 配列

### ドキュメント作成用テンプレート

```yaml
name: documentation
description: ドキュメント作成用テンプレート
tags:
  status: draft
  priority: low
  type: docs
  labels: [documentation]
```

### タスク管理用テンプレート（日付型を含む）

```yaml
name: task
description: 一般的なタスク用テンプレート
tags:
  status: todo
  priority: medium
  assigned_to: ""
  due_date: 2025-12-31
  start_date: 2025-01-01
```

### 作成手順:
- キー: `status`, 値: `todo`, 型: 文字列
- キー: `priority`, 値: `medium`, 型: 文字列
- キー: `assigned_to`, 値: (空), 型: 文字列
- キー: `due_date`, 値: `2025-12-31`, 型: **日付**
- キー: `start_date`, 値: `2025-01-01`, 型: **日付**

**日付型の使い方:**
- 型選択で「日付」を選択すると、日付ピッカーが表示されます
- YYYY-MM-DD形式（例: 2025-12-31）で保存されます
- 既存の日付値を編集する際も日付ピッカーが使用されます

## 6. テンプレートの編集

既存のテンプレートを変更する:

1. テンプレート一覧から編集したいテンプレートをクリックして展開
2. 「**編集**」ボタンをクリック
3. タグを追加/削除/変更:
   - **追加**: 下部の「新しいタグを追加」セクションから追加
   - **変更**: 既存タグの値フィールドを直接編集
   - **削除**: タグの右側の「削除」ボタンをクリック
4. 「**更新**」ボタンで保存

## 7. テンプレートの削除

不要なテンプレートを削除する:

1. テンプレート一覧から削除したいテンプレートを展開
2. 「**削除**」ボタンをクリック
3. 確認ダイアログで「OK」をクリック

⚠️ **注意**: 削除したテンプレートは復元できません!

## 8. テンプレートの適用（将来実装予定）

現在、テンプレートセレクター機能は実装済みですが、エディタへの統合はまだ完了していません。

### 予定されている機能:

1. **新規タスク作成時**
   - タスク作成ダイアログに「テンプレート選択」ボタンが表示
   - テンプレートを選択すると、自動的にフロントマターが挿入される

2. **既存タスク編集時**
   - エディタのツールバーに「テンプレート適用」ボタンが表示
   - マージモード/上書きモードを選択可能

## トラブルシューティング

### テンプレートが表示されない

- ワークスペースが正しく開かれているか確認
- ブラウザの開発者ツール（F12）でエラーがないか確認
- アプリを再起動してみる

### テンプレートが保存されない

- テンプレート名が空でないか確認
- 少なくとも1つのタグが追加されているか確認
- ワークスペースディレクトリの書き込み権限があるか確認

### デフォルトテンプレートが適用されない

- この機能はまだ実装されていません（将来実装予定）
