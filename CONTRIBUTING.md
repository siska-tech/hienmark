# HienMark への貢献

HienMarkプロジェクトへの貢献に興味を持っていただき、ありがとうございます！このドキュメントでは、プロジェクトへの貢献方法について説明します。

## 🚀 はじめに

### 開発環境のセットアップ

1. **必要な環境**
   - Node.js 20.x以上
   - Rust 1.77以上
   - Visual Studio Build Tools（Windows）/ Xcode（macOS）/ GCC（Linux）

2. **リポジトリのクローン**
   ```bash
   git clone https://github.com/yourusername/hienmark.git
   cd hienmark
   ```

3. **依存関係のインストール**
   ```bash
   npm install
   ```

4. **開発サーバーの起動**
   ```bash
   npm run tauri dev
   ```

詳細は [docs/DEVELOPMENT.md](docs/DEVELOPMENT.md) を参照してください。

## 📋 貢献の方法

### イシューの報告

バグを見つけた場合や新機能の提案がある場合は、GitHubのIssueを作成してください。

**バグ報告のテンプレート:**
```markdown
## 問題の説明
[バグの簡潔な説明]

## 再現手順
1. [手順1]
2. [手順2]
3. [手順3]

## 期待される動作
[期待される結果]

## 実際の動作
[実際に起こった結果]

## 環境
- OS: [例: Windows 11]
- HienMark バージョン: [例: 0.1.0]
- Node.js バージョン: [例: 20.10.0]
- Rust バージョン: [例: 1.90.0]
```

**機能リクエストのテンプレート:**
```markdown
## 機能の説明
[提案する機能の簡潔な説明]

## 動機
[なぜこの機能が必要か]

## 提案する実装
[どのように実装すべきか（任意）]

## 代替案
[検討した他の方法（任意）]
```

### プルリクエストの提出

1. **フォークとブランチ作成**
   ```bash
   git checkout -b feature/your-feature-name
   ```

2. **変更を実装**
   - コーディング規約に従ってコードを記述
   - 適切なコメントを追加
   - 必要に応じてテストを追加

3. **コミット**
   ```bash
   git add .
   git commit -m "feat: Add your feature description"
   ```

   **コミットメッセージの規約:**
   - `feat:` - 新機能
   - `fix:` - バグ修正
   - `docs:` - ドキュメントのみの変更
   - `style:` - コードの意味に影響しない変更（フォーマット等）
   - `refactor:` - リファクタリング
   - `test:` - テストの追加・修正
   - `chore:` - ビルドプロセスやツールの変更

4. **プッシュとPR作成**
   ```bash
   git push origin feature/your-feature-name
   ```

   その後、GitHubでPull Requestを作成してください。

**PRのテンプレート:**
```markdown
## 変更の概要
[変更内容の簡潔な説明]

## 関連Issue
Closes #[Issue番号]

## 変更の種類
- [ ] バグ修正
- [ ] 新機能
- [ ] ドキュメント更新
- [ ] リファクタリング
- [ ] その他（説明してください）

## テスト
- [ ] 既存のテストがすべてパス
- [ ] 新しいテストを追加
- [ ] 手動テストを実施

## チェックリスト
- [ ] コードが規約に従っている
- [ ] ドキュメントを更新した
- [ ] 変更が後方互換性を保っている
- [ ] コミットメッセージが規約に従っている
```

## 🎨 コーディング規約

### TypeScript / React

- **スタイル:** ESLintの設定に従う
- **コンポーネント:** 関数コンポーネントとHooksを使用
- **型定義:** `any`の使用を避け、明示的な型定義を行う
- **ファイル命名:** PascalCase（コンポーネント）、camelCase（ユーティリティ）

### Rust

- **スタイル:** `rustfmt`でフォーマット
- **命名:** snake_case（関数、変数）、PascalCase（型、構造体）
- **エラーハンドリング:** `Result`型を使用し、適切なエラーメッセージを返す
- **ドキュメント:** 公開API には必ずdocコメントを追加

### コード構成

```
HienMark/
├── src/                    # フロントエンド
│   ├── components/         # Reactコンポーネント
│   ├── services/           # バックエンドAPIクライアント
│   ├── hooks/              # カスタムフック
│   ├── types/              # TypeScript型定義
│   └── utils/              # ユーティリティ関数
├── src-tauri/              # バックエンド
│   └── src/
│       ├── commands/       # Tauriコマンド
│       ├── service/        # ビジネスロジック
│       ├── parser/         # パーサー
│       └── models/         # データモデル
└── docs/                   # ドキュメント
```

## 🧪 テスト

現在、テストカバレッジは限定的です。テストの追加は大歓迎です！

### テストの実行

```bash
# フロントエンドテスト（実装予定）
npm test

# Rustテスト（実装予定）
cd src-tauri
cargo test
```

## 📚 ドキュメントの更新

## 🔍 レビュープロセス

2. レビューコメントに対して修正を行ってください
3. すべてのチェックがパスし、承認されたらマージされます
